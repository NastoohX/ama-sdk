Chassis Setup¶

  1. BIOS setting:

    1. PCIe bifurcation must be enabled on each slot with a MA35 card.

    2. Secure boot must be disabled.

    3. Enable virtualization.

  2. Enable IOMMU by updating /etc/default/grub file to include

    • GRUB_CMDLINE_LINUX_DEFAULT="quiet splash amd_iommu=on iommu=pt", for AMD based systems

    • GRUB_CMDLINE_LINUX_DEFAULT="quiet splash intel_iommu=on iommu=pt", for Intel based chassis

  3. Update Grub

    sudo update-grub

  4. Allocate at least 8GB of huge pages

    sudo sh -c "echo 'vm.nr_hugepages=4096' >> /etc/sysctl.conf"

  5. Reboot

    sudo reboot

  6. Verify settings

    cat /proc/cmdline
``BOOT_IMAGE=/vmlinuz-5.15.0-50-generic root=/dev/mapper/vg00-rootlv ro quiet splash amd_iommu=on iommu=pt``

    and

    cat /proc/meminfo | grep -i huge

    , should return:

    HugePages_Total:    4096
HugePages_Free:     4096
HugePages_Rsvd:        0
Hugepagesize:       2048 kB

Check Cards¶

To establish whether deployed cards have proper firmware installed, proceed as follows:

  1. List AMD AMA Video SDK PCIe devices

    sudo lspci -vvvd 10ee:

    If devices are not listed as Multimedia controller and/or Region 4 size is not 512MB, a flash update is required. Contact AMD support team for instructions on how to proceed with updating your cards.

Install the AMD AMA Video SDK¶

  • Ensure that you PPA feed is setup. See Package Feed Setup

  • Ubuntu

    1. Install packages

      sudo apt -y install libhugetlbfs0 libboost-all-dev
dpkg --get-selections '*ma35*' | awk '{system("sudo apt -y purge " $1)}'
sudo apt update
sudo apt install ma35-base ma35-apps ma35-firmware ma35-ffmpeg ma35-libs ma35-utils

    2. Verify packages

      sudo dpkg -l ma35*

    Ensure that the return package names, match the ones used in Step 1.

Chassis Bringup¶

After each reboot or SDK update execute the following command:

  1. Insert kernel module

    sudo modprobe ama_transcoder

  2. Confirm driver’s successful installation by issuing:

    cat /sys/class/misc/ama_transcoder0/version_information

    It should return:

    <<<Version Info>>>
ZSP Version = 0.1.8
SC Version = 9.7.2
eSecure Version = 1.0.0
PCIe FW Version = 2.1.0
PCIe CTRL Patch Version = 1.0.1
PCIe PHY Patch A Version = 1.0.0

If the observed version numbers are not as the above, contact AMD support team for instructions on how to proceed with updating your cards.

Set Up the Runtime Environment¶

  1. Run setup script

    source /opt/amd/ma35/scripts/setup.sh
