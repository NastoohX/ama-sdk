AMD Advanced Media Acceleration (AMA) Release Notes¶

Note

Version: 0.3 (July Early Access)

Table of Contents

  • Overview

  • Release Requirements

  • Features in This Release

  • Known Limitations

  • Known Issues

Overview¶

This software is a limited, non-public, pre-release build supporting AMD AMA Video SDK MA35D card. The software is intended to demonstrate specific video processing capabilities as outlined in the Features section of this document and is supplied solely to demonstrate the capabilities enumerated in the Features section and is not intended nor suitable for deployment or utilization in any other manner.

Release Requirements¶

  1. The following firmware versions are required:

    1. Satelite Control (SC): 9.7.3

    2. ZSP: 0.1.15

  2. This documentation is applicable to July pre-release software.

Features in This Release¶

  1. Support for video transcode using either our supplied FFmpeg (n5.1), Gstreamer (v1.22), or custom Xilinx Media Acceleration (XMA) applications.

  2. Support for up to 4 x 4Kp60 8-bit or 10-bit video decode of either AVC or HEVC per device.

  3. Support for up to 2 x 4Kp60 8-bit or 10-bit video transcode of AVC or HEVC per device.

  4. Support for up to 4 x 4Kp60 8-bit or 10-bit video transcode of AV1 per device.

  5. Support for integration with software-based video image filters using our hardware DMA support to/from device memory (FFmpeg or Gstreamer).

  6. Optional use of our supplied resource management daemon (xrmd) for automated accelerator selection and load balancing.

  7. Resource utilization reporting when using our supplied resource management daemon (xrmd).

  8. Utility applications enabling collection of power, voltage, temperature and memory for each device as well as aggregate values for the MA35D accelerator card.

  9. Support for virtualization using either Ubuntu 20.04 or 22.04 in KVM mode with a guest OS also running either Ubuntu 20.04 or 22.04.

Known Limitations¶

  1. Full device scalability is supported to resolutions down to 720p60. Lower resolutions are supported but may not reach intended scalability targets.

  2. Support for 2D graphics engine and ML engine not supported in kernel driver.

  3. This software has been tested on Ubuntu 20.04 and Ubuntu 22.04 with kernel version 5.15.0.

  4. Host servers must support a BAR4 PCIe memory region of 512 MB

  5. Only a single VF may be assigned to a VM.

  6. For encoding, only resolutions that are divisible by 4 are supported, i.e., both height and width.

  7. For decoding, only resolutions that are divisible by 2 are supported, i.e., both height and width.

  8. Density targets for ULL transcoding at 4Kp60 for 8 or 10-bit, where both AV1 types are deployed, may not meet real time performance. Reducing overall channel density and/or utilizing one AV1 type at a time will ensure latency performance.

  9. A device that has been assigned to a VM cannot share its resources with its host.

Known Issues¶

  1. PCIe Gen5 links will be set to operate in PCIe Gen4 speeds.

  2. Host servers must support a BAR4 PCIe memory region of 512 MB.

  3. MA35D may not be able to ingest streams created by the MA35D.

  4. This release only supports encoding to rawvideo for AV1.

  5. HDR10 support is not available in this release.

  6. Multi-board installations (>4 boards) run at high channel counts may see high kernel CPU utilization.

  7. A device function level reset (FLR) or secondary bus reset (SBR) will cause the server to hang. A cold boot of the server will restore function.

    Workaround:

    For Linux based systems hosting the PF, it is recommended to disable PF FLR and SBR by writing to the following sysfs node:

    echo ‘ ‘ >/sys/bus/pci/devices/<domain-bus-device-function>/reset-method

    This will cause all PF FLR and SBR requests to the MA35D devices to be ignored.

  8. AV1 decoding is not supported in the reference XMA transcoder application (ma35_transcoder_app)

  9. AV1 decoding using the included ama_av1dec Gstreamer plugin requires the use of an IVF packaged AV1 raw video stream and a corresponding ivfparse plugin to be utilized to demux the AV1 video stream.

  10. If the kernel driver is not loaded, tools such as mautil may crash when invoked

  11. An AV1 HLS stream may playback at slower than real time speeds. Recommend explicitly setting of fps playback frame rate when using ffplay or use ffplay from FFmpeg 5.1.2 or 6.0

  12. AV1 muxted into an MP4 container may not play back properly. Recommend playback of raw video using more current versions of ffplay (n5.1.2 or later) or more recent versions of media players such as VLC

  13. An assertion or generic software error may result from running a transcode operation repeatedly over many iterations. After repeated execution of numerous transcode operations, a FIFO or other such software error may occur.

  14. When XRMD is run repeatedly, it can halt unexpectedly.

Upgrading from Previous Versions¶

  1. Remove any ma35 and amd-ama packages as per installation instructions.

  2. Remove any manually created Boost library soft links that may have been created, previously.
